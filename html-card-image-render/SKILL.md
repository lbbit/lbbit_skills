# html-card-image-render

## 目的

参考 `mermaid-markdown-image-render` 的思路，将“内容输入 -> 浏览器渲染 -> 图片输出”流程封装为可复用 skill。

本 skill 的目标是：
- 快速生成可用于 PPT / 文档 / 社媒封面的卡片图
- 支持多输入格式：`HTML`、`JSON`、`Markdown`、`TXT`
- 默认提供高可读模板，降低手写 CSS 的成本

## 目录结构

```text
html-card-image-render/
├── SKILL.md
├── package.json
├── puppeteer-config.edge.windows.json
├── scripts/
│   └── render_card_image.js
├── examples/
│   ├── architecture_card.json
│   ├── code_card.md
│   └── dashboard_card.html
└── verify_output/
```

## 适用场景（示例）

- PPT 架构总览卡片（SoC 模块、业务系统拓扑、组件关系）
- 高级数据看板卡片（KPI、同比环比、阈值状态）
- 代码讲解卡片（核心片段 + 要点解读）
- 方案对比卡片（A/B 方案、优缺点、成本对比）
- 发布说明卡片（版本亮点、改动摘要、风险提示）

## 输入建议

优先级建议：
1. 复杂视觉布局：直接使用 `HTML`
2. 结构化业务内容：使用 `JSON`
3. 快速草稿和说明：使用 `Markdown/TXT`

### JSON 推荐字段

```json
{
  "eyebrow": "ARCHITECTURE",
  "title": "典型嵌入式 SoC 内部结构",
  "subtitle": "CPU / GPU / NPU / DSP 协同",
  "description": "适用于演示层面的高密度信息可视化。",
  "tags": ["SoC", "AXI", "SRAM", "DRAM"],
  "metrics": [
    { "label": "模块数", "value": "8" },
    { "label": "总线协议", "value": "AXI/AHB" }
  ],
  "bullets": [
    "CPU 负责通用控制与调度",

  "code": {
    "language": "pseudo",
    "content": "schedule(cpu, gpu, npu, dsp)\\nroute(bus=AXI)"
  },
  "footer": "Generated by html-card-image-render"
}
```

## 执行方式

先安装依赖：

```bash
$env:PUPPETEER_SKIP_DOWNLOAD="true"
npm install
```

渲染 JSON：

```bash
node scripts/render_card_image.js \
  --input examples/architecture_card.json \
  --output verify_output/architecture_card.png \
  --theme architect \
  --width 1800 \
  --height 1100 \
  --puppeteer-config puppeteer-config.edge.windows.json
```

渲染 Markdown：

```bash
node scripts/render_card_image.js \
  --input examples/code_card.md \
  --output verify_output/code_card.png \
  --theme sunset \
  --puppeteer-config puppeteer-config.edge.windows.json
```

渲染 HTML（全页面）：

```bash
node scripts/render_card_image.js \
  --input examples/dashboard_card.html \
  --output verify_output/dashboard_card.png \
  --selector .dashboard \
  --puppeteer-config puppeteer-config.edge.windows.json
```

## 参数说明

- `--input`：输入路径（`.html/.htm/.json/.md/.txt`）
- `--output`：输出图片路径（默认 `output/card.png`）
- `--width` / `--height`：视口尺寸（默认 `1600x900`）
- `--device-scale-factor`：渲染倍率（默认 `2`）
- `--theme`：模板主题（`architect|sunset|forest|minimal`）
- `--selector`：元素截图选择器（默认 `#card-root`）
- `--format`：`png|jpeg|webp`（不传则根据输出后缀推断）
- `--quality`：JPG/WEBP 质量（`1-100`）
- `--full-page`：是否全页截图
- `--timeout`：超时时间（毫秒）
- `--puppeteer-config`：浏览器配置（可指向 Edge）

## 图片质量规范（建议）

- PPT 16:9 页面建议：`1920x1080` 或 `1600x900`
- 文档插图建议：宽度不低于 `1400`
- 开启 `--device-scale-factor 2` 提升清晰度
- 控制文本层级：主标题 <= 16 个字，副标题 <= 28 个字
- 每张卡片信息块建议 3~7 个，避免视觉拥挤

## 输出约定

成功输出：
- `[OK] Rendered image:`
- `- <absolute output path>`

失败输出：
- `[ERROR] ...`

## 注意事项

- 首次使用需要联网安装 npm 依赖。
- 推荐显式传入 `--puppeteer-config`，避免无浏览器环境导致失败。
- 若传入完整 HTML 页面，本脚本不会改写其样式，仅负责截图。
